📘 Quick Guide

30-Second Quickstart

  1. Type your name in Crew Member (only first time).
  2. Tap Tag #, enter the number on the keypad, hit Done.
  3. Pick Ranch, Pasture, Lot, Action. Most auto-fill.
  4. Confirm meds (auto-filled from your protocol — adjust dose if needed).
  5. Tap Save Record. Done.

🔒 Lock Fields for Chute Work

When working many head in the same pasture / lot / action, tap the 🔓 button next to Ranch, Pasture, Lot, and Action. The lock turns green. From then on, those fields stay filled in across saves — you only need to enter Tag and meds.

Ranch and Pasture are linked. Locking Pasture auto-locks Ranch. Unlocking Ranch auto-unlocks Pasture. Pasture can't be locked alone.

When any lock is active, a green banner up top shows what's locked. The keypad also auto-opens after each save.

🏷 Tag Field Buttons

  • ↺ ditto: Repeat the most recently saved tag.
  • NT: No-tag entry. Auto-numbers as NT1, NT2, NT3…
  • ✖ X: Clear the tag.

On a desktop or device with a hardware keyboard, the custom keypad is skipped — just type the tag and press Enter.

📜 Tag History

When you enter a tag with prior records, the alert box shows "⚠️ History: N previous records — tap to view ▸". Tap to see every prior record: action, date, meds, location, crew member, drug-off status. Dead records have a red border.

☁ Sync Status Badge

The badge next to 🔄 Pull Cloud History tells you what's happening:

  • No badge: Everything is synced.
  • ⏳ N pending: Records waiting to upload. Auto-retries every minute.
  • ⏳ Syncing N…: Currently uploading.
  • ⚠ Offline: No signal. Records save locally and sync when you reconnect.

Records save instantly even with no signal. Save then edit before the queue clears, only the latest version reaches the cloud.

📊 Daily Summary

The card at the top shows today's running tally — head treated, breakdown by action, time of last save. Auto-hides when there are no records yet.

📖 History Tab

View today's and yesterday's records. Edit loads a record back into the form (preserves the original treatment time). Del shows a pulsing red Confirm and a gray Cancel — nothing happens until you pick one. Auto-cancels after 5 seconds.

🧹 Storage Self-Healing

When local storage fills up after months of use, the app auto-prunes records older than 60 days — your spreadsheet keeps the archive. You'll get a toast warning when it happens.

💡 Tips

  • Add the app to your home screen for full offline use.
  • Pull Cloud History at the start of each day with signal.
  • Lock your fields before starting a chute session.
  • Press and hold any button for half a second to see what it does.

🛠 Troubleshooting

"My record isn't in the spreadsheet"

Most likely no signal at the time. Check the badge next to Pull Cloud History:

  • ⚠ Offline → Get back to signal. Records auto-sync.
  • ⏳ N pending → Queued. Retries every minute.
  • No badge → Synced. If still missing in the sheet, see below.

Don't tap Pull Cloud History to fix this — that downloads, not uploads. (Pulling is safe though.)

"Queue isn't clearing even with signal"

  • Toggle airplane mode on, then off.
  • Force-quit the app and reopen.
  • If still stuck, the cloud script may need attention from whoever set it up.

"Deleted records came back after Pull Cloud History"

Shouldn't happen any more — the app uses tombstones to remember deletes. If you see this, you're on a cached old version. Clear Safari website data and re-add the app to your home screen.

"Keypad won't open when I tap Tag #"

Use a quick tap — under half a second. A long press shows a tooltip instead. If still stuck, force-quit and reopen.

"Action is locked but meds aren't filling in"

  • Save at least one record with that action first.
  • Pull Cloud History to refresh the protocols list.
  • Make sure the action name in your spreadsheet exactly matches the dropdown option.

"I locked Pasture but Ranch unlocked itself"

By design — Pasture can't be locked without Ranch. Locking Pasture auto-locks Ranch; unlocking Ranch auto-unlocks Pasture. To work different pastures within the same ranch, lock just Ranch (tap Ranch's lock without touching Pasture's).

"Missing: Crew Member, Tag, Action, Ranch" toast

You hit Save with required fields empty. The toast names which ones — fill them in and try again.

"SAFETY CHECK FAILED" message

  • "Already has a 1st Pull" → Pick 2nd Pull instead, or edit the existing record.
  • "Already has a 2nd Pull" → Edit or delete the existing record if wrong.
  • "2nd Pull without a 1st Pull" → Record the 1st Pull first, or check the tag.
  • "Already marked Dead" → Verify the tag number.

"App froze"

Force-quit (swipe up, swipe the card away). Reopen. Your data is safe — local records and the queue both survive restarts.

"Pasture dropdown is empty"

That ranch has no pastures in the cloud sheet. Add them in the spreadsheet, then Pull Cloud History.

"Save button does nothing"

  • A required field is empty. Scroll up and check.
  • Or you're inside the 600ms cooldown after a previous tap. Wait a second.

"App looks blank or broken"

Bad cache from a partial update. Settings → Safari → Clear History and Website Data, then re-open and re-add to home screen. Pull Cloud History will restore your records from the sheet.

"Pull Cloud History says ❌ Data Error"

The cloud script returned something unexpected. Wait a few minutes and try again. If it keeps failing, the spreadsheet's Apps Script needs attention.

"Medication I need is missing"

Add it to the medications tab in the spreadsheet, then Pull Cloud History. For one-off use, just type the med name directly into the field — it doesn't have to match the dropdown.

"Storage full — pruned N old records" warning

Records older than 60 days were auto-removed from local storage to make room. The cloud spreadsheet still has them. No action needed.

If you see "🛑 Storage full. Pull Cloud, then clear app data." — Pull Cloud History to confirm everything's synced, then clear Safari website data and re-add the app to your home screen.

🚨 Emergency: Out in pasture and nothing works

Use a notebook. Write down: tag, ranch, pasture, lot, action, meds with doses, your name, time. Type them into the app that evening with Wi-Fi. The doctoring is more important than the app.

📞 When reporting a problem

Include: what you tapped right before, what the sync badge said, whether you had signal, how many records that day, screenshot if possible.
